# Moat and Defensibility Analysis

Use this skill as the primary operating playbook for this domain.

## Domain Focus
- Moat source mapping
- Time-to-replicate analysis
- Defensibility investment roadmap

## Workflow
1. Clarify target customer, painful problem, and why now.
2. Test problem-solution fit with lightweight experiments and direct signals.
3. Define the growth and monetization path with constraints and dependencies.
4. Prioritize the next highest-leverage bets for the next 30 to 90 days.
5. Package the narrative and proof for customers, partners, and investors.
6. Track learning velocity and kill low-signal initiatives quickly.

## Deliverables
- Founder memo with thesis and go-forward recommendation
- Validation plan with measurable pass fail criteria
- Execution roadmap for the next operating window
- Narrative assets for customers and investors

## Quality Bar
- Tie recommendations to measurable outcomes and explicit constraints.
- Make tradeoffs explicit and name what is intentionally deprioritized.
- Keep assumptions testable and define what signal would invalidate the plan.
- Produce outputs another agent can execute without additional clarification.
