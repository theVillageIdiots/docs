# Jobs To Be Done and Persona Modeling

Use this skill as the primary operating playbook for this domain.

## Domain Focus
- Job story construction
- Persona realism and anti-stereotypes
- Decision-useful segmentation

## Workflow
1. Define user segment, desired behavior change, and north-star outcome.
2. Gather evidence from research, analytics, and existing product signals.
3. Synthesize opportunities and prioritize by impact, effort, and risk.
4. Design the solution path with clear hypotheses and instrumentation.
5. Specify implementation details for design, copy, and engineering handoff.
6. Evaluate outcomes and iterate based on observed user behavior.

## Deliverables
- Problem framing document with user and business context
- Prioritized solution options with rationale
- Ship-ready requirements and UX guidance
- Experiment and measurement plan

## Quality Bar
- Tie recommendations to measurable outcomes and explicit constraints.
- Make tradeoffs explicit and name what is intentionally deprioritized.
- Keep assumptions testable and define what signal would invalidate the plan.
- Produce outputs another agent can execute without additional clarification.
