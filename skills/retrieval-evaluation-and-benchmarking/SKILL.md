# Retrieval Evaluation and Benchmarking

Use this skill as the primary operating playbook for this domain.

## Domain Focus
- Benchmark dataset design
- Metric suite selection
- Failure-mode leaderboard analysis

## Workflow
1. Define retrieval and reasoning use cases and target quality thresholds.
2. Design the conceptual model, entities, relationships, and constraints.
3. Establish ontology and taxonomy governance for consistency over time.
4. Implement indexing, linking, and retrieval features with traceability.
5. Evaluate precision, recall, ranking quality, and failure cases.
6. Iterate model and retrieval stack based on measured relevance gaps.

## Deliverables
- Conceptual and logical model documentation
- Governance rules for schema and vocabulary changes
- Evaluation suite with relevance metrics
- Improvement backlog prioritized by retrieval impact

## Quality Bar
- Tie recommendations to measurable outcomes and explicit constraints.
- Make tradeoffs explicit and name what is intentionally deprioritized.
- Keep assumptions testable and define what signal would invalidate the plan.
- Produce outputs another agent can execute without additional clarification.
